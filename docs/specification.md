📄 Техническое задание: Telegram как файловое хранилище (MVP)

## Глоссарий терминов

- **Канал** — публичный или приватный Telegram-канал, выбранный для хранения
  файлов.
- **Топик** — отдельная тема (thread) внутри Telegram-канала, соответствует
  одной локальной папке.
- **Папка** — директория на локальном диске пользователя, синхронизируемая с
  топиком.
- **Пользователь** — единственный пользователь приложения, обладающий всеми
  правами (админки нет).

---

🧭 Цель

Создать веб-приложение, позволяющее использовать Telegram-канал как хранилище
файлов, с возможностью загрузки файлов из локальных папок в топики Telegram, а
также выгрузки файлов из Telegram обратно на диск. Пользователь управляет связью
между локальными папками и топиками Telegram, следит за статусами синхронизации
и может переименовывать топики.

⸻

🔧 Стек технологий • Frontend: Next.js (App Router) + TailwindCSS,
аутентификация через NextAuth.js • Backend: Node.js (v24+) + WebSocket
(Socket.IO) • TypeScript: latest (tsconfig.base.json общая конфигурация для Node
и браузера) • ORM: Prisma • БД: SQLite • Telegram API: MTProto через gramJS •
Файловая система: Node.js fs, chokidar • Хранение сессии Telegram: в SQLite •
Docker сеть: default bridge, маппинг портов 3000/4000

- Валидация env: использовать dotenv-schema (схема в docs/env.schema.json)
- TELEGRAM_CHANNEL_IDS передаётся как CSV-строка

🧑‍💻 Функции пользователя

1. Работа с локальными папками
   - **Что видит пользователь:**
   - Дерево локальных папок (от базовой директории из .env)
   - Для каждой папки — количество файлов (только дочерние, без вложенных)
   - **Что может сделать:**
   - Выбрать одну или несколько папок для загрузки
   - **Ограничения:**
   - Структура и количество файлов обновляются автоматически каждые 60 секунд

2. Загрузка папок в Telegram
   - **Что видит пользователь:**
   - Список выбранных папок и соответствующих топиков
   - Статус загрузки и счётчик (x из y)
   - Сообщения об ошибках (например, конфликт имён)
   - **Что может сделать:**
   - Запустить загрузку одной или нескольких папок
   - Переименовать топик до или после загрузки
   - **Ограничения:**
   - Имя топика по умолчанию = путь до папки (относительно базовой директории)
   - Загружаются только новые файлы (по имени и расширению)
   - Файлы загружаются по одному, без архивирования
   - Если топик с таким именем уже существует — загрузка блокируется
   - Переименование обновляет имя топика в Telegram и в БД

3. Синхронизация с Telegram
   - **Что видит пользователь:**
   - Список каналов (если их несколько)
   - Для выбранного канала — список топиков и файлов
   - **Что может сделать:**
   - Просматривать структуру канал → топики → файлы
   - **Ограничения:**
   - По умолчанию используется один канал (ID из .env), но поддерживается работа
     с несколькими (список каналов в БД)
   - При запуске приложения загружаются все топики Telegram-канала
   - Если топик отсутствует в БД — создаётся новая запись
   - Если первое сообщение в топике — текст, оно используется как путь до папки,
     иначе путь = ""

4. Выгрузка из Telegram
   - **Что видит пользователь:**
   - Список топиков и файлов в каждом топике
   - Статус и прогресс выгрузки
   - Сообщения об ошибках (например, файл уже существует)
   - **Что может сделать:**
   - Выбрать топик для выгрузки
   - Указать родительскую папку
   - Переименовать папку перед скачиванием
   - Выбрать часть файлов для выгрузки (по паттерну или списку имён)
   - **Ограничения:**
   - Внутри выбранной папки создаётся папка с именем топика
   - Файлы с таким же именем уже существующие в папке не перезаписываются
   - Если выбран паттерн или список файлов — выгружаются только они, остальные
     игнорируются

### ⚙️ Технические детали

Telegram API (GramJS + MTProto): • Авторизация через api_id и api_hash из .env.
• Используется Telegram user session. • Сессия сохраняется в SQLite. • Если
Telegram API вернёт ошибку авторизации — автоматически переавторизуемся (в
фоне). • Ретрай с экспоненциальным бэкоффом: старт 10 сек, максимум 1 час, без
ограничения числа попыток; UI поддерживает команды `pause` и `stop` для
управления операциями

Канал(ы) • Структура: channels (таблица БД) • id, name, telegram_id, slug, и др.
• Возможность выбора канала из списка. • Если один канал — выбор отключён,
только отображение.

⸻

🛠️ Архитектура

Сервисы 1. FSService • Сканирует локальную папку (через chokidar) • Возвращает
дерево папок + количество файлов • Обновление каждую минуту 2. SyncService •
Отвечает за сопоставление папка ↔ топик • Обрабатывает загрузку файлов в
Telegram • Проверяет дубликаты • Хранит статус загрузки 3. TelegramService •
Инициализация сессии • Работа с каналами и топиками • Загрузка и выгрузка файлов

4. StorageService • Хранит связи папка ↔ топик, список каналов, Telegram-сессию
   • Работа через Prisma/SQLite
5. UI/WebSocket Layer • Две вкладки: • Local => TG (локальные папки) • TG =>
   Local (топики) • Отображение дерева, статуса загрузки, ошибок • Коммуникация
   с бэкендом через WebSocket (Socket.IO) • Для выгрузки файлов из топика
   поддерживается фильтрация по паттерну или списку имён файлов

### REST API (дополнение)

- `POST /api/download` { topicId, targetPath, pattern?: string, files?: string[]
  } → DownloadResult
- Ответы имеют формат:
  - Успех (200): `{ data: <тип>, code: 'success' }`
  - Ошибка клиента (4xx): `{ error: { code: string; message: string } }`
  - Ошибка сервера (5xx): `{ error: { code: string; message: string } }`

🧪 MVP Проверка и сценарии

Успешные сценарии: • Приложение загружается, Telegram-сессия активна, канал и
топики отображаются. • Пользователь выбирает локальную папку, указывает имя
топика, загружает в Telegram. • В UI отображается прогресс загрузки. •
Пользователь переименовывает топик — Telegram обновляется. • Пользователь
выбирает топик, выгружает файлы в локальную директорию.

Ошибочные сценарии: • Конфликт имён топика — показывается ошибка. • Telegram API
недоступен — статус “ожидание”, автоповтор. • При удалённой локальной папке —
папка в UI отображается как “не существует”.
