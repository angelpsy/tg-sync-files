# Обзор функциональности Telegram File Sync

## 📋 Общие сведения

**Telegram File Sync** — это экспериментальный инструмент для синхронизации
локальных файлов с Telegram, использующий мессенджер в качестве персонального
облачного хранилища. Основным механизмом организации данных являются **Topics**
(топики/форумы) внутри Telegram-каналов.

---

## 🛠 Ключевые возможности

### 1. Авторизация и безопасность

- **MTProto User-Session**: Приложение работает от имени обычного пользователя
  (не бота), что позволяет использовать все возможности Telegram, включая работу
  с форумами и загрузку файлов до 2 ГБ.
- **Интерактивный вход**: Поддержка ввода номера телефона, кода подтверждения и
  2FA пароля через веб-интерфейс.
- **Сохранение сессии**: Сессия сохраняется в SQLite, что исключает
  необходимость повторного входа после перезапуска сервера.

### 2. Локальное отслеживание (Local Monitoring)

- **Watcher (chokidar)**: Бэкенд в реальном времени следит за изменениями в
  базовой директории.
- **Сканирование**: Периодическое полное сканирование (каждые 30-60 сек) для
  актуализации состояния файлов, которые могли быть изменены во время остановки
  приложения.
- **Дерево папок**: Генерация метаданных о структуре папок для отображения в UI.

### 3. Синхронизация: Загрузка (Upload)

- **Folder-to-Topic Mapping**: Каждая локальная папка может быть привязана к
  отдельному топику в Telegram-канале.
- **Оркестрация загрузки**:
  - Параллельная загрузка нескольких файлов (настраиваемо).
  - Поддержка политик конфликтов (пропуск дубликатов, перезапись).
  - Вычисление хэшей файлов (MD5/SHA256) для проверки идентичности.
- **Автоматизация**: Возможность автоматического создания топика при начале
  синхронизации папки.

### 4. Синхронизация: Выгрузка (Download)

- **Просмотр контента**: Возможность просматривать список файлов внутри любого
  топика Telegram прямо из приложения.
- **Выборочное скачивание**: Пользователь может выбрать конкретные файлы или
  целые топики для загрузки на диск.
- **Фильтрация**: Поддержка паттернов поиска (glob) для массовой выгрузки только
  определенных расширений или имен.

### 5. Пользовательский интерфейс (Real-time Dashboard)

- **WebSockets (Socket.IO)**: Мгновенное обновление статусов загрузки,
  прогресс-баров и появления новых файлов без перезагрузки страницы.
- **Управление операциями**: Возможность ставить загрузки на паузу или полностью
  останавливать их.
- **Event Feed**: Лента событий, отображающая успехи и ошибки синхронизации.

## 🏗 Техническая архитектура

- **Frontend**: Next.js 15 (App Router), TypeScript, Tailwind CSS, shadcn/ui.
  Используется архитектура **Feature-Sliced Design**.
- **Backend**: Node.js, Express. Платформа построена на принципах **Clean
  Architecture**.
- **Telegram Integration**: Библиотека **GramJS**.
- **Database**: SQLite с использованием **Prisma ORM** для хранения метаданных
  синхронизации и сессий.

## ⚙️ Настройка (Environment)

Основные переменные в `.env`:

- `TELEGRAM_API_ID` / `TELEGRAM_API_HASH`: Ключи для подключения к API Telegram.
- `DATABASE_URL`: Строка подключения к SQLite.
- `WATCH_DIR`: Путь к папке на сервере, которую нужно синхронизировать.
- `BACKEND_WS_PORT`: Порт для WebSocket-сервера.
